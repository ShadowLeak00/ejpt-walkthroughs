Assessment 01 - ejpt Information gathering CTF (INE)

Hello guys…. Today I have brought you a detailed walkthrough for the very first CTF in ejpt course.

This CTF is based on information gathering phase. So, without further ado lets go team……

Here are the hints we got from INE

Flag 1: This tells search engines what to and what not to avoid.

  • When we read the above line the very first thing comes to our mind is the robot.txt file. This is the file we create rules for search engines about what it can get access and what cannot get access
  • Simply search http://target.ine.local/robots.txt URL
  • Yay…. Here we got the first flag

Flag 2: What website is running on the target, and what is its version?

  • To see the web technology and its version we have many options like
    • nmap -sV target.ine.local ⇒ nmap version scan
    • wappalyzer web extension
    • builtwith web extension
    • whatweb http://target.ine.local ⇒ Terminal tool in kali linux
  • Here I have used whatweb. And then yay… we got the second flag also

Flag 3: Directory browsing might reveal where files are stored.

  • For directory browsing or simply called as “Fuzzing” we have several command line tools as well as burpsuite options like,
    • burpsuite intruder directory fuzzing
    • dirb http://target.ine.local
    • ffuf -u http://target.ine.local -w /usr/share/wordlist/dirb/big.txt
  • Here I have used ffuf command line tool. We got some folder results
  • Now go through all the possible folders and file to find some juicy stuff
  • The thing I have found here wp-content path not showing anything. but I saw the images stored in /wp-content/…/assests/images… like path. Thats why I tried to use ffuf command again to the complete url called http://target.ine.local/wp-content/FUZZ
  • Then I got some folders I didnt see while browsing
  • Yep… We got the 3rd flag inside uploads folder

Flag 4: An overlooked backup file in the webroot can be problematic if it reveals sensitive configuration details.

  • They told us there may be a backup file with a sensitive data in a root directory. In usually backup files are ends with .bak, .tar, .tar.gz, .zip, .sql, .bak.zip like that
  • in ffuf command there is a option called “-e” to mention which extensions we want to search specially
  • ffuf -u http://target.ine.local/ -w /usr/share/wordlist/dirb/big.txt -e .bak,.zip,.tar,.tar.gz,.bak.zip
  • Now we got a special file called “wp-config.bak”
  • Now we want to see what data are inside the file but you cant see when browsing
  • so, we can use curl http://target.ine.local/wp-config.bak
  • We got the 4th flag inside this file

Flag 5: Certain files may reveal something interesting when mirrored.

  • We are using httrack command line tool in kali linux to mirror the complete website
  • So, we can go through all the files and folders offline
  • httrack http://target.ine.local -o target.ine.local ⇒ by this command we can download the whole website into mentioned folder
  • inside target.ine.local you can find a file called “xmlrpc0db0.php” something kind a juicy file
  • yay… Inside this file we got the last flag
